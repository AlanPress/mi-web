1. Inyección a Base de Datos (SQL Injection)

La inyección SQL ocurre cuando un atacante inserta código malicioso dentro de una consulta SQL. Esto sucede por falta de validación en los parámetros enviados a la base de datos.

Consecuencias:
  • Robo o destrucción de datos.
  • Acceso no autorizado.
  • Control total del servidor.
Ejemplo de ataque: 
' OR 1=1 --

2. Cross-Site Scripting (XSS)

XSS permite a un atacante ejecutar JavaScript malicioso dentro del navegador de otros usuarios. Esto ocurre cuando una aplicación muestra datos sin sanitizarlos.

Consecuencias:
  • Robo de cookies y sesiones.
  • Redirecciones maliciosas.
  • Modificación del contenido mostrado al usuario.
Ejemplo de ataque: 
<script>alert('XSS')</script>

3. Fallos de Validación

Los fallos de validación suceden cuando los datos ingresados por el usuario no son verificados antes de procesarse. Esto puede causar vulnerabilidades como inyección, XSS o corrupción de datos.

Errores comunes:
  • No validar tamaño, tipo o formato de datos.
  • Validar solo en el lado del cliente.
  • Permitir caracteres especiales sin control.

4. Manejo Inseguro de Sesiones

Un mal manejo de sesiones puede permitir que un atacante suplante la identidad de un usuario.

Problemas comunes:
  • IDs de sesión predecibles.
  • Sesiones sin expiración.
  • No usar cookies seguras (HttpOnly, Secure).
Riesgos:
  • Secuestro de sesión (Session Hijacking).
  • Robo de información sensible.
